Conceptos útiles

¿Qué hace un desarrollador web?​

Un desarrollador web es responsable de diseñar, crear y mantener sitios web y aplicaciones web. Esto implica trabajar con lenguajes de programación, como HTML, CSS, JavaScript y otros, para crear la estructura, el diseño y la funcionalidad del sitio web o aplicación.

Los desarrolladores web pueden trabajar en el desarrollo de la parte delantera (front-end) del sitio web, que incluye la interfaz de usuario y la experiencia del usuario, o en el desarrollo de la parte trasera (back-end) del sitio web, que se ocupa de la lógica del servidor y la gestión de bases de datos.

Los desarrolladores web también pueden trabajar en el desarrollo de aplicaciones web, que son programas que se ejecutan en línea y que permiten a los usuarios realizar tareas específicas, como la gestión de proyectos, la edición de imágenes, la gestión de redes sociales, entre otras. En este caso, el desarrollador web se encarga de programar la lógica de la aplicación, así como la interfaz de usuario y la experiencia del usuario.

En general, los desarrolladores web deben estar familiarizados con las últimas tecnologías y tendencias en el desarrollo web, así como con los requisitos de los clientes y las necesidades de los usuarios finales.

La web es un sistema de información que se encuentra en la red de redes de internet. Los navegadores web se comunican con los servidores web para obtener los documentos de la web. ¿Qué es HTML?​

HTML es un lenguaje de marcado que se utiliza para crear páginas web. HTML significa HyperText Markup Language (Lenguaje de Marcado de Hipertexto). HTML no es un lenguaje de programación, sino un lenguaje de marcado. HTML describe la estructura de una página web semánticamente junto con el texto, las imágenes y otros contenidos. HTML no define el diseño visual de una página web, para eso se utiliza CSS.

¿Qué es CSS?​

CSS significa Cascading Style Sheets (Hojas de Estilo en Cascada). CSS describe cómo se muestran los elementos HTML. CSS no controla el contenido de un documento HTML, sino que sólo define cómo se muestra.

¿Qué es JavaScript?​

JavaScript es un lenguaje de programación interpretado, dialecto del estándar ECMAScript. Se define como orientado a objetos, basado en prototipos, imperativo, débilmente tipado y dinámico.

¿Qué es Bootstrap?​

Bootstrap es un framework de código abierto para desarrollar sitios y aplicaciones web. Contiene plantillas de diseño con tipografía, formularios, botones, cuadros de diálogo, navegación y otros elementos de interfaz de usuario, así como extensiones de JavaScript opcionales.

¿Qué es Git?​

Git es un sistema de control de versiones diseñado por Linus Torvalds, pensando en la eficiencia y la confiabilidad del mantenimiento de versiones de aplicaciones cuando estas tienen un gran número de archivos de código fuente. Su propósito es llevar registro de los cambios en archivos de computadora y coordinar el trabajo que varias personas realizan sobre archivos compartidos.

¿Qué es GitHub?​

GitHub es una plataforma de desarrollo colaborativo de software para alojar proyectos utilizando el sistema de control de versiones Git. Esencialmente, hace que sea más fácil para los individuos y equipos usar Git como la versión de control y colaboración.

¿Qué es un repositorio?​

Un repositorio es un espacio donde se almacenan los archivos de un proyecto. Los repositorios pueden contener carpetas y archivos, imágenes, vídeos, hojas de cálculo, y cualquier otro tipo de archivo que tu proyecto necesite.

¿Qué es un commit?​

Un commit es un punto en la historia de tu proyecto. Cada commit tiene un identificador único (llamado SHA) que lo identifica de forma única y consistente en tu repositorio, y un mensaje asociado que es escrito por la persona que hizo el commit. Los commits normalmente se usan para marcar puntos de referencia importantes en la historia de un proyecto, como cuando se resuelve un error, o cuando se añade una nueva característica.

¿Qué es un branch?​

Un branch es una línea de desarrollo independiente dentro de un repositorio. Cada repositorio tiene un branch por defecto llamado master, y se usa para almacenar el código de producción, listo para implementar. Puedes usar otros branches para desarrollar funcionalidades, solucionar errores, o hacer cualquier otra cosa que no esté lista para implementar.

¿Qué es un pull request?​

Un pull request es una solicitud para que los cambios que has hecho en un branch sean revisados y fusionados con el branch principal (por ejemplo, master). Cuando envías un pull request, estás pidiendo que alguien revise tus cambios y los fusionen con el branch principal. Puedes enviar un pull request desde tu propio branch o desde un fork de un repositorio.

¿Qué es un merge?​

Un merge es la acción de fusionar dos o más branches de Git. El merge resultante es un nuevo commit. Los merges se pueden realizar tanto desde la interfaz web de GitHub como desde la línea de comandos.

¿Qué es un fork?​

Un fork es una copia de un repositorio. Puedes hacer forks de repositorios para probar cambios sin afectar el branch principal del repositorio original. Los forks también te permiten contribuir a otros proyectos.

¿Qué es un issue?​

Un issue es un problema, una tarea o una característica que se quiere añadir al proyecto. Los issues se pueden crear tanto desde la interfaz web de GitHub como desde la línea de comandos.

¿Qué es un pull?​

Un pull es una acción que se realiza para actualizar el branch actual con los cambios que se han hecho en el branch remoto. Los pulls se pueden realizar tanto desde la interfaz web de GitHub como desde la línea de comandos.

¿Qué es un push?​

Un push es una acción que se realiza para actualizar el branch remoto con los cambios que se han hecho en el branch actual. Los pushes se pueden realizar tanto desde la interfaz web de GitHub como desde la línea de comandos.

¿Qué es un clone?​

Un clone es una copia local de un repositorio. Los clones se pueden realizar tanto desde la interfaz web de GitHub como desde la línea de comandos.

¿Qué es un stash?​

Un stash es una forma de guardar cambios temporales que no quieres hacer commit. Los stashes se pueden realizar tanto desde la interfaz web de GitHub como desde la línea de comandos.

¿Qué es un tag?​

Un tag es un punto en la historia de un repositorio. Los tags se usan para marcar puntos de referencia importantes, como una versión de lanzamiento (v1.0).
